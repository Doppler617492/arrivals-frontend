Strateški tehnološki sklop i ključne arhitektonske odluke

Ovo izvješće pruža sveobuhvatnu tehničku validaciju i poboljšanja predloženog plana za redizajn "Arrivals Dashboarda". Predloženi tehnološki sklop iznimno je moderan i robustan te se temelji na dokazanim praksama za razvoj aplikacija koje se oslanjaju na podatke. Kombinacija Reacta i TypeScripta osigurava pouzdan, skalabilan i dobro strukturiran temelj za razvoj korisničkog sučelja. Odabir Vitea kao alata za izgradnju, zajedno s TanStack Queryjem za upravljanje podacima i TanStack Tableom za složene prikaze, jasno pokazuje usmjerenost na performanse i učinkovito programiranje.

1.1 Arhitektura poslovne nadzorne ploče: model jednostranične aplikacije (SPA)

Predložena arhitektura jednostranične aplikacije (SPA) je optimalna za internu, autentificiranu i vrlo interaktivnu aplikaciju. UI se učita jednom, a prijelazi su brzi i fluidni. SSR/SSG (npr. Next.js) primarno koristi javni web (SEO/TTFB), što ovdje nije prioritet.

1.2 Detaljna analiza i opravdanost tehnološkog sklopa

Vite naspram Webpacka

Vite koristi ESM i pruža vrlo brzo pokretanje i HMR. Webpack je robustan, ali kompleksniji i sporiji za HMR u većim aplikacijama. Module Federation nije potreban.

shadcn/ui (Radix)

shadcn/ui je distribucija koda – komponente su u vlasništvu projekta, bez vendor lock‑in‑a. To osigurava dugoročnu prilagodljivost.

2. UI/UX, sustav dizajna i pristupačnost

Paleta (Indigo, Cyan, Green, Amber, Rose, Slate), Inter font, Tailwind + shadcn/ui. Teme preko CSS varijabli. A11y po WCAG 2.1 AA (kontrast ≥ 4.5:1, fokus, alt tekst, tipkovnica).

3. Sloj podataka i API

REST je trenutno optimalan. Za budućnost, GraphQL na /graphql reducira over/under‑fetching i round‑tripove.

4. Vizualizacija i izvještavanje

ECharts za grafove visokih performansi. SheetJS za CSV/XLSX izvoz. TanStack Table za interaktivne tablice.

5. Performanse i skalabilnost

Lazy loading, virtualizacija, memoizacija, background refresh u TanStack Query (staleTime).

6. Testiranje i CI/CD

Vitest + RTL (unit/komponentno), Playwright (E2E), Lighthouse ciljevi u CI. Playwright nudi napredne mogućnosti (više preglednika, multi‑context).

7. Preporučeni API uzorci

Parametri: from, to, tz, group, filters, plus sort, pagination i opcijski fields za selekciju polja.
